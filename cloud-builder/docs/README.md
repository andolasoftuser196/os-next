## Configuration

The cloud builder requires database and cache services (PostgreSQL, Redis, etc.) to be available. These can be:

- **Hosted locally** (Docker, native installations)
- **Cloud-hosted** (AWS RDS, ElastiCache, etc.)
- **Third-party services** (managed databases, etc.)

Configure the connection details in the deployment package `.env` file.

**Q: How do I build the binary?**

**Q: How do I deploy?**
- Docker: `cd dist-docker && docker compose up -d`
- Native: `cd dist-native && ./run.sh`

**Q: Where are deployment packages?**
- Docker: `dist-docker/` (auto-generated by build.py)
- Native: `dist-native/` (auto-generated by build.py)

**Q: How do I update configuration?**
- Edit in `orangescrum-cloud-common/config/`
- Rebuild packages: `cd orangescrum-cloud-docker && ./build.sh`

**Q: How do I troubleshoot issues?**
- Docker: `cd dist-docker && docker compose logs orangescrum-app`
- Native: Check application output
